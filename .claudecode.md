# Claude Code Project Guidelines

As the AI agent for this repository, you must adhere to the following rules for every task.

## 1. Coding Philosophy
- **Readability Over Optimization:** Prioritize clean, easy-to-read, and maintainable code over "clever" optimizations or raw execution efficiency. Use descriptive variable names and clear logic structures.
- **Styling:** Use **Tailwind CSS** for all styling tasks. Avoid custom CSS files or inline style objects unless Tailwind cannot achieve the required result.

## 2. Permissions & Workflow
- **Read Operations:** You have permanent, explicit permission for ALL read operations. Do not ask for confirmation before reading files, searching the directory, or inspecting the codebase.
- **Deletions:** You MUST ask for explicit human permission before deleting any file or significant blocks of code.
- **Verification:** Before reporting a task as "Done," you must execute `npm run build` to ensure the code compiles and builds correctly. If the build fails, fix the errors and retry before notifying the user.

## 3. Quality Control
- **Linting:** Always run the project's linting command (e.g., `npm run lint` or `eslint .`) after modifying code. Fix any linting errors automatically before finishing the task.
